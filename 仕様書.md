# 講義ノート自動生成アプリ 仕様書

本アプリは2つの機能からなる。

- **本体（バッチ）**: 講義を録音し、停止すると自動で文字起こし→階層ノート化までを行う Mac 常駐アプリ（§1〜§4, §7〜§10）。
- **追加（リアルタイム字幕）**: 録音中に「声を拾えているか」を目で確認できるライブ字幕を表示する。精度は粗くてよく、最終ノートは本体バッチが担う（§5）。本体パイプラインは変更しない純粋な追加レイヤー。

## 1. 目的

- **本体**: 講義を録音し、停止すると自動で文字起こし→階層ノート化までを行う。手作業（録音→mlx-whisper→手書き整形）を、ボタン1つに置き換える。
- **ライブ字幕**: 録音中に「ちゃんと声を拾えているか」を**目で確認できる安心感**を提供する（速報の文字起こし＋音声レベル表示）。

## 2. 構成（決定事項）

| 項目 | 採用 | 理由 |
|------|------|------|
| 形態 | メニューバー常駐アプリ（Electron + TypeScript） | 普段使いの操作性。既存スタックを活用 |
| 録音 | ffmpeg（avfoundation でマイク入力）、16kHz mono **WAV（可逆）** | 導入済み。Whisper内部は16kHz化するため高サンプルレート不要。AAC非可逆を避け精度優先 |
| 文字起こし（本体） | mlx-whisper `large-v3-turbo` | 実機検証で `large-v3-mlx`(非turbo) は空文字起こしになる不具合を確認し turbo を採用。精度は turbo＋固有名詞 initial-prompt＋要約時のclaude文脈補正で担保 |
| 要約 | `claude -p`（既定）／Ollama（代替）の**差し替え式** | 既定はサブスク範囲で追加課金ゼロ。将来の有料化に備え切替可能に |
| 保存形式 | Markdown（見出し＋本文＋箇条書き＋表を使い分けた読みやすいノート） | 箇条書き一辺倒は見づらいため、構造化された読みやすい形式にする |
| ライブ字幕 | 録音中だけ表示する常時最前面・フレームレスのウィンドウ（字幕＋音声レベル） | 目的が「動作確認」なので最小実装で直結。メニューバー運用を壊さない |
| ライブ文字起こし | 軽量 mlx_whisper モデル（既定 `whisper-base-mlx`）を**常駐 Python** に保持し、数秒セグメントを逐次処理 | CLIは毎回モデルロードで不可。粗くてよいので軽量モデルで十分・完全オフライン |

## 3. 処理フロー（本体）

```
[トレイ:録音開始]
   → ffmpeg でマイクを録音（録音中はトレイアイコンが赤）
   → （録音中は並行してライブ字幕経路が走る。詳細は §5）
[トレイ:録音停止]
   → ① mlx_whisper で文字起こし（.txt）
   → ② 末尾/途中の幻聴ループを除去（整形）
   → ③ claude -p に文字起こしを渡し、階層ノート（.md）を生成
   → ④ 保存（録音/文字起こし/ノートを1フォルダに）
   → ⑤ 完了通知＋ノートを開く
```

## 4. 機能要件（本体）

### 4.1 録音
- トレイメニュー「録音開始 / 停止」
- 入力デバイスは**名前で指定して起動時に番号へ解決**（avfoundation のデバイス番号は変動するため）。既定: `MacBook Airのマイク`
- 録音中の経過時間をトレイに表示
- 出力: `録音.wav`（16kHz mono, 可逆）
- **信頼性（codex指摘を採用）**:
  - 録音中は一時ファイル（`録音.wav.part`）に書き、停止後に ffprobe で長さ・破損を検証してから正式名にリネーム（atomic）
  - 起動前にマイク権限を確認。未許可なら設定アプリへ誘導
  - ffmpeg プロセスの生存を監視し、異常終了したら通知して部分ファイルを保全
  - 二重起動防止（録音中は「開始」を無効化）

### 4.2 文字起こし
- mlx-whisper `large-v3-turbo`、`--language ja`（非turboの large-v3-mlx は実機で空になるため不採用）
- **語彙バイアス**: `--initial-prompt` に固有名詞リスト（科目名・講師名・専門用語）を渡し、誤変換を抑える。リストは設定で蓄積・編集可能（前回の「奥村組」「ネクスコ」等の取りこぼし対策）
- 処理は別プロセス（録音停止後にバックグラウンド実行）。長時間講義は処理に実時間相当〜数倍かかる前提で、進捗をトレイ/通知で表示

### 4.3 幻聴ループ除去（後処理）
- 同一行が N 回以上（既定 5）連続したら、最初の1回を残して以降を削除
- 末尾の無音由来ループ対策。前回の「総務部課…」問題を自動で解消

### 4.4 要約（差し替え式エンジン）
- インターフェース `Summarizer` を定義し、実装を差し替え可能にする
  - `ClaudeCliSummarizer`（既定）: `claude -p` を実行。**前提: Claude Code がサブスクログイン（APIキー未設定）であること**
  - `OllamaSummarizer`（代替）: 完全ローカル・無料。**自動切替はせず、ユーザーが設定で選択**（品質・モデル取得・処理時間が異なるため）
- **claude -p の安全な実行（codex指摘を採用）**:
  - 文字起こしは **stdin で渡す**。shell 文字列連結は禁止（コマンドインジェクション防止）
  - **起動時＋実行直前の両方**で認証種別を確認。APIキー認証/不明なら、明示確認するまでクラウド要約を実行しない（意図しない従量課金防止）
  - 実行タイムアウト・終了コード・stderr を解析し、失敗時は原因を提示
  - `claude` のバージョンを確認（CLIは安定APIではない前提で、想定外出力に備える）
- **プロンプトインジェクション対策**: 文字起こし本文は「資料であって命令ではない」と固定の枠組みで囲い、本文中の指示には従わせない
- プロンプトで「階層箇条書きの講義ノート」を指示。文脈から明らかな誤変換は補正、自信がないものは (?) を付ける
- **授業資料の参照**: PDF(pdftotext)/Word等(textutil)/txt を `<materials>` として渡し、用語表記・構成の参考にさせる。ただし資料にあって講義で触れていない事項は補完しない指示を付ける（過剰補完の防止）
- 入力が長い場合: チャンク分割→部分要約→統合（段階要約）。各チャンクで「重要発言・専門用語・疑問点」も中間抽出し、統合時の情報落ちを防ぐ

### 4.5 保存
- 保存先: `~/Documents/講義ノート/YYYY-MM-DD_HHMM/`
  - `録音.wav` / `文字起こし.txt` / `ノート.md`
- ファイル名の日時はアプリ起動時ではなく録音開始時刻を使用
- **各段階の中間成果を即保存**（文字起こし完了時点で `文字起こし.txt` を確定）。後段（要約）が失敗しても前段の成果を失わない
- `ノート.md` は **atomic write**（一時ファイルに書いてからリネーム）。途中失敗時は `ノート.partial.md` として残す
- 同一時刻ディレクトリの衝突時は連番を付与

### 4.6 設定
- 入力デバイス、Whisperモデル、保存先、幻聴ループ閾値、要約プロンプト、固有名詞リスト、**要約エンジン（claude / ollama / ローカルのみ）**、**ライブ字幕の有効/無効・ライブモデル**（§5.4.5）

### 4.7 ジョブ状態管理（codex指摘を採用）
- 1回の処理を `Recording → Transcribing → Cleaning → Summarizing → Done / Failed` の状態機械として管理
- 各段階でキャンセル可能、失敗時はその段階からリトライ可能
- 失敗通知は「失敗」だけでなく、原因・保存済み成果物・再試行導線を提示

### 4.8 依存コマンドの確認（codex指摘を採用）
- 起動時に `ffmpeg` / `mlx_whisper` / `claude`（または `ollama`）の存在とバージョンを確認
- ライブ字幕有効時は Python(mlx_whisper) とライブモデルの取得状況も確認（§5.4.5）
- 未インストール時はインストール手順を案内

## 5. リアルタイム字幕（ライブ文字起こし表示）

### 5.1 目的・スコープ

**目的**: 録音中に「ちゃんと声を拾えているか」を**目で確認できる安心感**。録音中だけ画面に **ライブ字幕（速報の文字起こし）＋音声レベル** を表示する。

**スコープの原則（最重要）**
- **最終ノートの品質が本命**。ライブ字幕は速報用で**精度は粗くてよい**。
- 既存のバッチ処理（停止後の `large-v3-turbo` 文字起こし → 整形 → 要約 → `ノート.md` 保存）は **コードもファイルも一切変えない**。
- ライブ字幕は**純粋な追加レイヤー**。失敗してもバッチ処理・録音本体に影響しない（独立性）。
- **完全オフライン**で動く（ライブ字幕にクラウドを使わない。`HF_HUB_OFFLINE=1` ＋ ライブモデル事前DL）。

**やらないこと（YAGNI）**
- 単語単位ストリーミング（whisper の `word_timestamps` はメモリリークがあるため使わない。§5.4.2）。
- 字幕のドラッグ移動・リサイズ・テーマ設定（v1 は固定表示）。
- 字幕テキストの保存（ライブ字幕は揮発。記録は従来の `文字起こし.txt`/`ノート.md` が担う）。

### 5.2 全体構成

```
[録音開始]
   │
   ├─ ① ffmpeg（1プロセス・多出力）  ← 既存 Recorder を拡張
   │     ├─ 出力A: 録音.wav.part        … 連続WAV（従来と完全同一。バッチ用）
   │     ├─ 出力B: live/seg_%03d.wav     … 数秒セグメント（ライブ字幕用・揮発）
   │     └─ 出力C: astats → stdout       … 音声レベル（onLevel 指定時のみ）
   │
   ├─ ② LiveTranscriber（TS）           ← 新規
   │     ├─ live/ を監視し新規 seg_*.wav を検知
   │     ├─ ③ 常駐Python（mlx_whisper, 軽量モデル）に stdin でパスを渡す ← 新規
   │     │     └─ stdout に文字起こしJSONを返す（モデルは初回のみロード）
   │     ├─ 処理済みセグメントを削除
   │     └─ 確定テキストを caption ウィンドウへ push
   │
   └─ ④ 字幕ウィンドウ（Electron BrowserWindow）← 新規
         └─ 受け取ったテキスト・レベルを表示

[停止/失敗/異常終了]
   → ④破棄・②停止・③終了 を必ず実行（全経路で）
   → 以降は既存 stopAndProcess のバッチ処理（無改変）
```

**コンポーネント境界**

| 単位 | 役割 | 依存 | 新規/変更 |
|---|---|---|---|
| `Recorder`（`src/pipeline/record.ts`） | ffmpeg 多出力の起動・停止・レベル抽出 | ffmpeg | **変更**（出力B/C追加） |
| `LiveTranscriber`（`src/pipeline/liveTranscribe.ts`） | セグメント監視・常駐Python管理・テキスト供給 | 常駐Python | **新規** |
| 常駐スクリプト（`scripts/live_transcribe.py`） | モデル常駐・短クリップ文字起こし | mlx_whisper | **新規** |
| 字幕UI（`assets/caption.html` / `assets/caption.js`） | 表示専用 | preload.cjs | **新規（静的ファイル）** |
| `electron/main.ts` | ライフサイクル配線・IPC・設定 | 上記すべて | **変更**（配線追加） |
| バッチ系（`transcribe.ts` / `run.ts` / `summarize` 他） | 最終ノート生成 | — | **無変更** |

### 5.3 処理フロー（録音中のライブ経路）

1. `startRecording()` 成功直後に **字幕ウィンドウ生成** ＋ **LiveTranscriber 起動**。
2. ffmpeg が `live/seg_000.wav`, `seg_001.wav` … を約5秒ごとに書き出す。
3. LiveTranscriber が新規セグメントを検知 → 常駐Python に絶対パスを stdin で渡す（最新番号は書き込み中の可能性があるため、次が現れるまで処理しない。停止時のみ最後の1個も処理）。
4. Python が文字起こし結果を JSON 1行で stdout に返す → 字幕ウィンドウへ push（直近 N 件をローリング表示）。
5. 処理済みセグメント `.wav` を削除（ディスク逼迫防止）。
6. `stopAndProcess()` で字幕ウィンドウ破棄・LiveTranscriber停止・Python終了・`live/` 掃除。**その後の既存バッチ処理は無改変。**

> 遅延は「セグメント長(5s) + 文字起こし処理(数百ms〜) + 初回モデルロード」。目的（動作確認）には十分。

### 5.4 機能要件

#### 5.4.1 音声取り込み（ffmpeg 多出力）

**ffmpeg コマンド**（実機検証済み。`record.ts` の spawn 引数を拡張。出力Bは `segmentDir` 指定時、出力Cは `onLevel` 指定時のみ付与）:

```
ffmpeg -f avfoundation -i :<index> \
  -map 0:a -ac 1 -ar 16000 -f wav -y <out>.wav.part \
  -map 0:a -ac 1 -ar 16000 -f segment -segment_time 5 -reset_timestamps 1 \
     -segment_format wav -y <segdir>/seg_%03d.wav \
  -map 0:a -af "asetnsamples=n=4800:p=1,astats=metadata=1:reset=1:\
     measure_perchannel=none:measure_overall=Peak_level+RMS_level,\
     ametadata=mode=print:file=-:direct=1" -f null -
```

**確定事項（R1 実機検証）**
- 単一入力 `:index` が複数出力へ暗黙 fan-out。**マイクを2プロセスで開く必要はない**。
- 出力A（連続WAV）は**既存と完全同一**。`stop()` の stdin `q` graceful finalize・ffprobe検証・rename・`detached:true` は**無改修で流用**。
- `q` 停止で出力A/Bとも finalize（連続WAVのRIFFサイズが実バイト数にpatchされ、既存 `probeDurationSeconds` を通る）。セグメントの**最後の半端断片（実測0.5s）も捨てられない**。
- セグメント連番 `%03d` は999超で自動的に4桁以上へ拡張（桁あふれの心配なし）。固定名は muxer エラーになるため連番必須。

**制約**
- C1: セグメント出力先 `live/`（録音先 `outDir` の下）は**録音本体と別ディレクトリ**にする。単一プロセスゆえ片出力の write 失敗で全体 abort しうるため、巻き添えで本体録音を失わない配置にする。
- C2: 処理済みセグメントは**逐次削除**する。16kHz/mono/s16 は約1.9MB/分。
- C3: 極端に短いセグメント（既定 16000B 未満 ≒ 0.5s）は**スキップ**（無駄な文字起こし呼び出しと幻覚を減らす）。
- C4: `detached:true` は維持必須（既存のCtrl-C対策）。
- C5（**未実装・任意**）: 停止フェイルセーフ（`q` で一定時間内に閉じなければ `SIGTERM`→`SIGKILL`）。現状は既存の `q` 停止のまま（実機ではマイクで `q` は確実に効く）。

#### 5.4.2 ライブ文字起こし（mlx_whisper 常駐ヘルパー）

**方式（R2 実機検証）**: `mlx_whisper.transcribe()` は内部 `ModelHolder`（クラス変数）でモデルをプロセス内キャッシュする。同じ `path_or_hf_repo` を渡し続ければ**ロードは初回のみ**。よって CLI を毎回起動するのではなく、**Python常駐スクリプト**を1本立て、stdin でクリップパスを受け、stdout に結果JSONを返す（最初に `{"ready":true}` を出して常駐準備完了を通知）。

**常駐スクリプト `scripts/live_transcribe.py`（要点）**
```python
# argv[1]=モデルrepo, argv[2]=initial_prompt（固有名詞バイアス・任意）
for line in sys.stdin:                      # 1行 = WAVクリップの絶対パス
    r = mlx_whisper.transcribe(
        path, path_or_hf_repo=REPO, language="ja",
        word_timestamps=False,             # メモリリーク回避（必須）
        condition_on_previous_text=False,  # クリップ独立・幻覚伝播抑止
        initial_prompt=initial_prompt, verbose=False,
    )
    sys.stdout.write(json.dumps({"text": r["text"].strip()}, ensure_ascii=False)+"\n")
    sys.stdout.flush()
```

**確定事項・制約（R2 実機検証）**
- L1: **モデルは `whisper-base-mlx` を既定**（速度/精度バランス。最速優先なら `whisper-tiny-mlx`）。最終ノートの `large-v3-turbo` とは**別プロセス・別モデル**。
- L2: `word_timestamps=False` 必須（`True` は1反復あたり約10MB増の既知リーク #1254）。
- L3: **完全オフライン**は `HF_HUB_OFFLINE=1`（常駐Python起動時に付与）＋ **ライブモデルの事前DL**で担保。未DLだとライブ字幕は自動OFF（§5.4.5）。
- L4: **Pythonインタプリタの絶対パス解決**が必要。PATH の `python3` に mlx_whisper が無いことがある。`which mlx_whisper`（console_script）の**shebang から正しい python を解決**、フォールバックで `python3`。
- L5: stdout はクリーンJSONのみ。`verbose=False` でも tqdm 進捗が **stderr** に出る → main側は stderr を**データ扱いせず**破棄。`PYTHONUNBUFFERED=1` ＋ 明示 flush で即時化。
- L6: **無音/非音声で定番幻覚**（「ご視聴ありがとうございました」等）が出る。緩和：C3の短断片スキップ ＋ `condition_on_previous_text=False`。**既知フレーズの簡易フィルタは未実装（任意）**。粗精度許容のため軽い対策で可。
- L7: **mlx-whisper のバージョンに依存**（`ModelHolder` は未文書化の実装詳細）。READMEに前提を明記。
- L8: 既存 `record.ts` の WAV 形式（pcm_s16le/16000Hz/mono）をそのまま `transcribe()` に渡せる（実測成功）。

#### 5.4.3 字幕ウィンドウ（Electron）

**方式（R4 検証）**: 録音中だけ表示する別 `BrowserWindow`。main→renderer の一方向 push のみ（invoke 不要）。

**ウィンドウ生成（確定パラメータ）**
- `frame:false, transparent:true, hasShadow:false, resizable:false, movable:false, skipTaskbar:true, focusable:false, show:false`
- 生成後: `setAlwaysOnTop(true,"screen-saver")`、`setVisibleOnAllWorkspaces(true,{visibleOnFullScreen:true,skipTransformProcessType:true})`、`setIgnoreMouseEvents(true,{forward:true})`（クリックスルー）
- 表示は `once("ready-to-show")` 内で `showInactive()`（フォーカスを奪わない）
- 位置: プライマリディスプレイ下中央（`screen.getPrimaryDisplay().workArea` から算出）

**確定事項・制約（R4 検証）**
- U1: `skipTransformProcessType:true` 必須（`app.dock.hide()` 済みアプリで dock アイコン再表示/ちらつき #25368 回避）。
- U2: `caption.html` / `caption.js` は **`assets/` に静的ファイルとして手書き**で置く（`settings.html`/`settings.js` と同じ。electron-builder の `files:["assets/**/*"]` で同梱）。
- U3: CSP は既存踏襲 `default-src 'self'; style-src 'unsafe-inline'` → inline script 不可、JSは外部 `.js`。`html,body{background:transparent}` 必須（透過窓の黒/灰背景バグ #40515 回避、`hasShadow:false` 併用）。
- U4: preload は既存 `assets/preload.cjs`（CJS）に受信用 `caption` API を追記（`onText`/`onLevel`、`removeListener` を返す）。
- U5: `webContents.send` は `isDestroyed()` ガード必須。`ready-to-show` 前の send は破棄される。
- U6: 表示は**確定セグメントテキストのローリング表示**（直近 N 件＝既定3件を連結）。「partial（未確定）」状態は持たない。

#### 5.4.4 音声レベルメーター

「声を拾えているか」を最も確実に示す指標。字幕テキストとは独立に動く。

**確定事項・制約（R3 実機検証）**
- M1: 5.4.1 のC出力（`astats`→stdout）を `onLevel` 指定時に付与。`measure_perchannel=none` 必須。
- M2: **stdout を必ず drain する**（`stdio` を `["pipe","ignore","pipe"]`→`["pipe","pipe","pipe"]`）。`detached` プロセスで stdout を読まないと**ffmpeg が完全ハングし `q` も効かず録音が死ぬ**（実測）。
- M3: 値抽出は `/(?:Peak|RMS)_level=(-?inf|-?[\d.]+)/`、`-inf`（無音時）を -90dB に丸める（`parseAstatsLevel`）。
- M4: 更新間隔は `asetnsamples` では正確に決まらない。**main側で約150msスロットル**して送信頻度を一定化。dBFS（-60〜0）を 0〜1 に線形マップ（`dbToLevel`）してバー表示。
- M5: レベルは録音パイプライン（ffmpeg）側で算出して push。renderer に音声ストリームは無いため Web Audio は不可。

#### 5.4.5 設定・依存・ライフサイクル

**設定追加（`AppConfig`）**
- `liveCaption: boolean`（既定 `true`）／`liveModel: string`（既定 `"mlx-community/whisper-base-mlx"`）
- 字幕の `initial_prompt` は既存 `vocabulary` を流用。設定画面に有効/無効チェックとモデル入力を追加。

**依存チェック（`deps.ts`）**
- ライブ字幕有効時：(a) `import mlx_whisper` 可能な Python を解決（L4）、(b) `liveModel` がHFキャッシュに存在（snapshots 配下が非空）。
- 起動時にライブ依存が欠けていれば「ライブ字幕は使えません（録音・ノートは通常どおり）」と別枠通知。設定画面の「依存コマンドを確認」にも `live: python` / `live: model` を表示。
- モデル未取得・Python不在・実行時例外は**ライブ字幕だけ自動 OFF にして録音は続行**（本体機能を止めない）。

**ライフサイクル（`main.ts` 配線）**
- 生成: `startRecording()` の `recorder.start()` 成功後（早期 return より後）。
- 破棄: `stopAndProcess()` ＝**停止・失敗・異常終了の全経路**で必ず字幕窓破棄＋LiveTranscriber停止＋Python終了＋`live/` 掃除。
- 状態機械（`idle|recording|processing`）は既存のまま流用。

## 6. 非機能要件（全体）

**本体**
- オフライン動作（要約の claude -p / ollama 利用時を除く）
- 録音データはローカル保存。クラウド送信は要約時の文字起こしテキストのみ
- **プライバシー（codex指摘を採用）**: 講義音声・文字起こしは機微情報になり得る。初回起動時に「クラウド要約を使う／ローカルのみ」を明示選択させる。既定でいきなり外部送信しない
- 長時間（2時間程度）の講義でもメモリ・処理が破綻しないこと
- 保存ファイルはユーザー権限のみ。本文をログ/クラッシュレポートに混入させない

**ライブ字幕**
- **既存非改変**: バッチ系ファイル（`transcribe.ts`/`run.ts`/`summarize.ts`/`clean.ts`/`chunk.ts`/`prompt.ts`）と連続WAV出力は不変。
- **独立性**: ライブ字幕の例外・Pythonクラッシュ・モデル未DLは録音/バッチに波及しない（try/catch で隔離、失敗時はライブのみ停止）。
- **プライバシー/オフライン**: ライブ字幕は端末内処理のみ（クラウド送信なし）。出ていく通信はライブモデルの取得時のみ。
- **メモリ**: 長時間（2時間級）でも常駐Pythonが破綻しない（L2のリーク回避）。
- **ディスク**: セグメントは逐次削除（C2）。録音終了時に `live/` を掃除。
